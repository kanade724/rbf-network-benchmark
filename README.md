# rbf-network-benchmark

[English](README.md) | [中文](README_cn.md)

A configurable benchmark framework for Gaussian RBF network evaluation on classical datasets, with future hardware-aware simulation support.

## Project Goal

This project evaluates a classic Radial Basis Function network on four datasets:

- Wine dataset
- Breast Cancer Wisconsin dataset
- Iris dataset
- Fashion-MNIST

The current iteration focuses on software evaluation. A later iteration can add hardware-aware RBF simulation without changing the basic project layout.

## Directory Design

The repository contains code and configuration only. Datasets and experiment outputs are written to the outer SURF2026 workspace.

```text
SURF2026/
  data/                         # downloaded datasets, generated by download_datasets.py
  output/                       # training and testing outputs
  src/
    rbf-network-benchmark/
      config.yaml               # all configurable parameters
      requirements.txt
      src/
        download_datasets.py    # downloads datasets into SURF2026/data
        train_evaluate.py       # trains and evaluates the RBF network
```

This keeps datasets and generated results out of the code repository.

## RBF Network Structure

The model is implemented in `src/train_evaluate.py`.

The hidden layer uses the classic Gaussian RBF:

```text
phi(x, c) = exp(-||x - c||^2 / (2 * sigma^2))
```

In code, this appears in `RBFNetwork._gaussian_rbf_features`:

```python
hidden = np.exp(-squared_distances / (2.0 * self.sigma_**2))
```

The network structure is:

```text
input features -> Gaussian RBF hidden layer -> softmax output layer
```

RBF centers are selected by either KMeans or random training samples. By default, the output layer is trained with softmax gradient descent, which records loss and accuracy values for each epoch. The training loop uses `tqdm` to show per-dataset progress.

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## Configuration

All configurable parameters are in `config.yaml`.

Important sections:

- `paths`: controls workspace, data, and output directories.
- `download.datasets`: controls which datasets are downloaded.
- `experiment.datasets`: controls which datasets are trained and tested.
- `preprocess`: controls feature extraction and standardization.
- `rbf`: controls the default RBF network parameters.
- `dataset_overrides`: controls per-dataset parameter overrides.

Each dataset can override RBF training parameters such as `epochs`, `learning_rate`, and `l2_alpha` through `dataset_overrides`:

```yaml
dataset_overrides:
  iris:
    rbf:
      epochs: 150
      learning_rate: 0.05
      l2_alpha: 0.001
  fashion_mnist:
    rbf:
      epochs: 150
      learning_rate: 0.05
      l2_alpha: 0.01
```

The default path setup is:

```yaml
paths:
  workspace_root: ../..
  data_dir: data
  output_dir: output
```

Because this repository is under `SURF2026/src/rbf-network-benchmark`, `../..` resolves to the `SURF2026` workspace root.

## Download Datasets

Download all configured datasets:

```powershell
python src\download_datasets.py
```

Download only one dataset:

```powershell
python src\download_datasets.py --dataset iris
python src\download_datasets.py --dataset wine
python src\download_datasets.py --dataset breast_cancer
python src\download_datasets.py --dataset fashion_mnist
```

Fashion-MNIST is fetched from OpenML, so the first download requires internet access.

## Train and Test

Train and evaluate all configured datasets:

```powershell
python src\train_evaluate.py
```

Train and evaluate one dataset:

```powershell
python src\train_evaluate.py --dataset iris
```

Outputs are written to:

```text
SURF2026/output/rbf_YYYYMMDD_HHMMSS/
```

Each dataset gets:

- `metrics.json`
- `classification_report.txt`
- `predictions.csv`
- `training_curves.csv`
- `loss_curve.csv`
- `accuracy_curve.csv`
- `loss_curve.png`
- `accuracy_curve.png`
- `confusion_matrix.csv`
- `confusion_matrix.png`

The full run also gets:

- `summary.json`

## Notes on Fashion-MNIST

Fashion-MNIST is image data, so the default config uses HOG features before the RBF layer. This keeps edge and shape information that raw pixel distances often miss:

```yaml
dataset_overrides:
  fashion_mnist:
    preprocess:
      feature_extractor: hog
```

This keeps the classifier as an RBF network while giving it image-oriented input features.
